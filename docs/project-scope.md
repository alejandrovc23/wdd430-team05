# Handcrafted Haven Project Scope

## Project Purpose

Handcrafted Haven connects artisans and crafters with customers seeking unique handmade goods. The platform is intended to support artisans and creative communities by making their work and stories easier to discover. It will encourage sustainable consumption, help customers explore distinctive products, and give each artisan space to share the people, skills, and processes behind their work.

## Intended Users

- Customers looking for unique handmade goods
- Visitors exploring products and artisan stories
- Artisans who want to present their work and creative identity
- Sellers who need to manage profiles and product listings
- Future application administrators who will help maintain the marketplace

## Functional Requirements

The complete semester project is planned to support:

- Authenticated seller profiles
- Artisan biographies
- Artisan stories
- Product listings
- Product descriptions
- Product prices
- Product images
- A browsable product catalog
- Category filters
- Price filters
- Sorting controls
- Product ratings
- Written product reviews

## Design Requirements

- Responsive design that works across mobile, tablet, and desktop screens
- A WCAG 2.1 Level AA accessibility goal
- Strong loading and runtime performance
- Valid, standards-based markup and styles
- Search-engine optimization (SEO)
- Clear and efficient usability
- Consistent branding throughout the experience
- Clear, predictable navigation

## Technology Requirements

- HTML for semantic content structure
- CSS for responsive presentation and the design system
- JavaScript for application behavior
- Next.js for the front-end application
- Node.js for server-side capabilities
- A database for persistent data in a later project phase
- Git for version control
- GitHub for repository hosting and collaboration
- GitHub Projects for backlog and workflow tracking
- Vercel for eventual deployment

## Week 02 Scope

Week 02 establishes the project foundation and planning materials. Its scope includes:

- Repository setup
- Local development setup
- Project planning
- A documented collaboration workflow
- An initial design identity
- A defined color palette
- Typography choices
- Responsive layout direction
- Initial reusable components
- Work-item brainstorming
- GitHub Project setup and backlog planning
- An initial Next.js foundation

The Week 02 interface and data are demonstrations of the planned direction, not a complete e-commerce implementation.

## Outside Week 02 Scope

The following work is explicitly outside the Week 02 scope:

- Full authentication
- Database integration
- A seller dashboard
- A product-management system
- A real shopping cart
- Checkout
- Payments
- A complete back end
- A complete testing suite
- A production release

## Current Limitations

All product, category, and artisan data is currently static. No database is connected, authentication is not implemented, and e-commerce transactions are not available. The initial architecture, components, and visual decisions may evolve in later weeks as requirements are refined and the team reviews the foundation.
