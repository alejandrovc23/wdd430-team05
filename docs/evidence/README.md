# Week 02 Evidence Guide

Capture the following screenshots manually after the related work has been completed and verified. Review every image before submission to ensure that it is readable and contains no passwords, tokens, environment variables, private messages, or other sensitive information.

## Screenshot 1 — Local Repository

Recommended filename: `01-local-repository.png`

It must show:

- Visual Studio Code.
- The `wdd430-team05` folder.
- The project files.
- `README.md` or `package.json` visible in the Explorer or editor.
- The current Git branch.
- No passwords or tokens.

## Screenshot 2 — Design Theme

Recommended filename: `02-design-theme.png`

It must show:

- The local `/design-system` page.
- The color palette.
- Typography examples.
- Button examples.
- A product-card example.
- Accessibility information.

Run the local application, open `http://localhost:3000/design-system`, and choose a viewport wide enough to make all required details readable.

## Screenshot 3 — GitHub Project Board

Recommended filename: `03-project-board.png`

It must show:

- The **Handcrafted Haven – Team 05** project title.
- The Board view.
- Status columns.
- At least ten work-item cards.

## Screenshot 4 — Work-Item Descriptions

Recommended filename: `04-work-item-descriptions.png`

It must show:

- At least one expanded issue.
- Its meaningful description.
- Its user story.
- At least three acceptance criteria.

These screenshots cannot be generated by the repository itself. They must be captured manually, reviewed for sensitive information, and uploaded with the individual Canvas submission.
